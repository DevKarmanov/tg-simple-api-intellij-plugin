Validates method parameter types for custom Telegram bot annotations.

This inspection verifies that methods annotated with Telegram bot annotations such as @BotText, @BotPhoto, @BotCallBack, @BotScheduled, and others have the correct method signature.

Specifically, it checks that the method parameters match the expected types depending on the annotation:

  • @BotText, @BotPhoto, @BotCallBack, @BotDocument, @BotLocation, @BotMedia, @BotVoice — exactly one parameter of type org.telegram.telegrambots.meta.api.objects.Update
  • @BotScheduled — exactly one parameter of type Set<Long>

If the method does not conform to these rules, the inspection reports an error with a clear message specifying the expected parameter type. 

            
                // Correct example:
                @BotText
                public void handleUpdate(Update update) { ... }

                // Incorrect example:
                @BotText
                public void handleUpdate(String text) { ... } // Error: must have one parameter of type Update

This helps prevent runtime errors and enforces consistency in your Telegram bot handler methods.
